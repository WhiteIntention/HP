# Assignment 1.4  
Вычисление среднего значения: последовательный и параллельный способы (OpenMP)

## Описание
В данном задании реализуется вычисление среднего значения элементов массива
двумя способами:
- последовательным
- параллельным с использованием OpenMP и механизма редукции
Проводится сравнение времени выполнения обеих реализаций.

## Условия задания
- Создать динамический массив из 5 000 000 чисел 
- Заполнить массив случайными значениями  
- Вычислить среднее значение элементов последовательным способом  
- Вычислить среднее значение с использованием OpenMP (reduction)  
- Сравнить время выполнения  
- Освободить выделенную память  

## Логика работы программы
1. Выделяется память под динамический массив  
2. Массив заполняется случайными числами  
3. Последовательно вычисляется сумма и среднее значение  
4. Измеряется время выполнения  
5. Параллельно вычисляется сумма с использованием OpenMP reduction  
6. Измеряется время выполнения  
7. Результаты выводятся в консоль  
8. Память освобождается  

## Файлы в папке
- `main.cpp` — исходный код программы  
- `screenshot.png` — результат работы программы (консоль)  
- `block.png` — блок-схема алгоритма  

## Компиляция и запуск
### GCC / MinGW (OpenMP)
g++ Ass1_task4.cpp -fopenmp -O2 -std=c++17 -o Ass1_task4.exe
.\Ass1_task4.exe   

## Анализ результатов
Последовательная и параллельная реализации выдают одинаковые значения суммы и среднего, что подтверждает корректность работы алгоритма. Параллельная версия с OpenMP в данном эксперименте оказалась медленнее из-за накладных расходов на создание потоков и операцию редукции, а также из-за ограниченной пропускной способности памяти.

## Вывод
Параллельное выполнение с OpenMP не всегда обеспечивает ускорение. Для задач с простыми операциями и интенсивным доступом к памяти последовательная реализация может работать быстрее.
