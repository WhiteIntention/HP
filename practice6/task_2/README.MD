# Task 2 (OpenCL): Parallel Matrix Multiplication

В данной работе реализована программа для **параллельного умножения матриц** с использованием технологии **OpenCL**.  
Программа вычисляет произведение двух матриц на GPU и сравнивает результат и время выполнения с последовательной реализацией на CPU.

## Описание задачи
Вычисляется произведение матриц:
- `A` размера **N × M**
- `B` размера **M × K**
- результирующая матрица `C` размера **N × K**

Операция: C = A × B

## Реализация
Программа включает:
- последовательную реализацию умножения матриц на CPU;
- параллельную реализацию на GPU с использованием OpenCL;
- измерение времени выполнения CPU и GPU;
- проверку корректности результатов.
Вычисления на GPU выполняются с помощью OpenCL-ядра, где каждый рабочий элемент отвечает за вычисление одного элемента матрицы `C`.

## Измерение производительности
Измеряются следующие величины:
- **CPU time** — время последовательного умножения матриц;
- **GPU kernel time** — чистое время выполнения OpenCL-ядра;
- **GPU total time** — общее время GPU-вычислений, включая передачу данных между host и device.

Эксперименты проведены для нескольких комбинаций размеров матриц, что позволяет оценить влияние размеров входных данных на производительность.

## Проверка корректности
Результаты, полученные на GPU, сравниваются с результатами CPU-реализации.  
Корректность подтверждается значением `Correct = YES`.  
Небольшая числовая погрешность (`Max error ~ 10⁻⁵`) является допустимой и связана с особенностями вычислений с плавающей точкой.

## Вывод
Использование OpenCL позволяет значительно ускорить умножение матриц по сравнению с последовательной реализацией на CPU, особенно при увеличении размеров матриц.  
Для малых размеров матриц выигрыш от GPU уменьшается из-за накладных расходов на передачу данных.

## Используемые технологии
- **C++**
- **OpenCL**
- **GPU (Tesla T4)**

## Контрольные вопросы:
### Основные типы памяти в OpenCL:
Global, Local, Private и Constant память.

### Настройка рабочих групп:
Глобальный размер задаёт число рабочих элементов, локальный — их группировку внутри рабочей группы.

### Отличие OpenCL от CUDA:
CUDA работает только на GPU NVIDIA, OpenCL — на разных устройствах и платформах.

### Преимущества OpenCL:
Переносимость, поддержка разных устройств и независимость от производителя.
