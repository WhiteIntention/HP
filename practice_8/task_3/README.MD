# Задание 8.3 — Гибридная обработка массива (CPU + GPU)

## Цель
Реализовать гибридное приложение, в котором обработка массива данных
выполняется одновременно на **CPU** и **GPU**, а также замерить общее
время выполнения такой обработки.

Операция обработки: **умножение каждого элемента массива на 2**.

---

## Что сделано
1. Исходный массив данных размера `N` создаётся на CPU.
2. Массив делится на две части:
   - первая половина обрабатывается на CPU с использованием OpenMP;
   - вторая половина обрабатывается на GPU с использованием CUDA.
3. CPU и GPU выполняют вычисления **параллельно**:
   - CPU работает со своей частью массива;
   - GPU асинхронно копирует данные, выполняет ядро и возвращает результат.
4. Измеряется **общее время гибридной обработки**, включая передачу данных и синхронизацию.

---

## Тестовые данные
Эксперименты проведены для двух размеров массива:
- `N = 1 000 000`
- `N = 10 000 000`

---

## Результаты

### N = 1 000 000
- First element (CPU part) = `1.392938`  
- First element (GPU part) = `0.831937`  
- Hybrid total time (ms) = `8.864407`  

### N = 10 000 000
- First element (CPU part) = `0.784892`  
- First element (GPU part) = `1.911649`  
- Hybrid total time (ms) = `18.764362`  

Вывод значений элементов подтверждает, что обработка выполнена корректно
как для CPU-, так и для GPU-части массива.

---

## Короткий анализ
- Гибридная обработка позволяет задействовать одновременно ресурсы CPU и GPU.
- Для небольших массивов выигрыш от гибридного подхода ограничен накладными
расходами на передачу данных и синхронизацию.
- Для больших массивов время выполнения определяется в основном GPU-частью
и копированием данных.
- Асинхронные операции и использование потоков позволяют частично перекрывать
вычисления на CPU и GPU.
