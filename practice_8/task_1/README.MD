# Задание 8.1 — Обработка массива на CPU с использованием OpenMP

## Цель  
Реализовать обработку массива данных на CPU с использованием **OpenMP** и замерить время выполнения.  
В качестве примера обработки используется операция: **умножить каждый элемент массива на 2**.

---

## Что сделано
- Созданы массивы случайных чисел размеров:
  - `N = 1 000 000`
  - `N = 10 000 000`
- Реализована функция обработки массива на CPU с директивой `#pragma omp parallel for`.
- Измерено время выполнения обработки с помощью `std::chrono`.
- Для простой проверки выводится первый элемент массива после обработки.

---

## Результаты

### N = 1 000 000
- First element = `1.392938`  
- CPU OpenMP time (ms) = `0.485274`

### N = 10 000 000
- First element = `0.784892`  
- CPU OpenMP time (ms) = `6.027850`

---

## Короткий анализ
- Время выполнения увеличивается при росте размера массива, что ожидаемо, так как количество операций линейно зависит от `N`.
- OpenMP позволяет распараллелить обработку массива по потокам CPU, поэтому даже для `10 000 000` элементов время остаётся небольшим.
- Для маленьких массивов ускорение может быть ограничено накладными расходами на создание/управление потоками, но при больших `N` параллелизм окупается.
