# Задание 9.2 — Распределённое решение СЛАУ методом Гаусса (MPI)

## Цель
Реализовать распределённое решение системы линейных уравнений `A x = b`
методом Гаусса с использованием MPI и продвинутых коллективных операций:
- распределение строк матрицы между процессами через `MPI_Scatter`;
- рассылка текущей опорной (pivot) строки через `MPI_Bcast` на каждом шаге прямого хода;
- сбор результатов на `rank = 0` через `MPI_Gather` и выполнение обратного хода.

---

## Идея реализации
1. Процесс `rank = 0` генерирует матрицу `A` размера `N × N` и вектор `b`.
2. Чтобы можно было проверить корректность, сначала генерируется известное решение `x_true`,
   затем вычисляется `b = A * x_true`.  
   Так можно сравнить найденное решение с эталоном.
3. Строки матрицы распределяются между процессами с помощью `MPI_Scatter`.
   Если `N` не делится на число процессов, добавляются “пустые” строки (padding),
   чтобы `MPI_Scatter` работал корректно.
4. Прямой ход:
   - процесс-владелец pivot-строки подготавливает её;
   - pivot-строка передаётся всем процессам через `MPI_Bcast`;
   - каждый процесс обновляет только свои строки (где глобальный индекс больше текущего `k`).
5. После прямого хода матрица и `b` собираются обратно на `rank = 0` (`MPI_Gather`),
   и `rank = 0` выполняет обратный ход и печатает решение.

---

## Анализ
Нулевая ошибка Max abs error vs x_true = 0.000000 означает, что распределённая реализация
корректно выполняет прямой ход и восстановление решения.
Совпадение первых элементов решения для N=128 и N=256 объясняется тем, что генератор
случайных чисел инициализируется фиксированным seed (123), поэтому первые элементы
x_true получаются одинаковыми при разных N.
Использование MPI_Bcast на каждом шаге прямого хода обеспечивает одинаковую pivot-строку
на всех процессах, что необходимо для корректного распределённого исключения.

алгоритма и правильное использование MPI_Scatter, MPI_Bcast и MPI_Gather.
