# Задание 9.1 — Распределённое вычисление среднего и стандартного отклонения (MPI) + время выполнения

## Цель
Реализовать распределённую обработку массива случайных чисел и вычислить:
- среднее значение,
- стандартное отклонение,

с использованием:
- `MPI_Scatterv` для корректного разбиения массива (включая случай, когда `N` не делится на число процессов),
- `MPI_Reduce` для сборки сумм и сумм квадратов,
- `MPI_Wtime()` для измерения времени выполнения и сравнения при разных `-np`.

---

## Краткая логика программы
1. На `rank = 0` создаётся массив случайных чисел размера `N`.
2. Массив делится между процессами через `MPI_Scatterv`.
3. Каждый процесс считает для своей части:
   - сумму элементов,
   - сумму квадратов элементов.
4. Суммы собираются на `rank = 0` через `MPI_Reduce`.
5. На `rank = 0` вычисляются:
   - `mean = sum / N`
   - `stddev = sqrt(E[x^2] - (E[x])^2)`
6. Печатаются результаты и время выполнения.

---

## Измерение времени
Время измеряется через `MPI_Wtime()` с синхронизацией процессов (`MPI_Barrier`).
Замер включает:
- распределение данных (`MPI_Scatterv`),
- локальные вычисления,
- агрегацию результатов (`MPI_Reduce`).

---

## Результаты (N = 1 000 000)

### np = 1
- Mean = `0.500094`
- StdDev = `0.288560`
- Execution time = `0.002087` seconds

### np = 2
- Mean = `0.500094`
- StdDev = `0.288560`
- Execution time = `0.001786` seconds

### np = 4
- Mean = `0.500094`
- StdDev = `0.288560`
- Execution time = `0.003222` seconds

---

## Анализ
- Значения `Mean` и `StdDev` совпадают для всех `-np`, что подтверждает корректность разбиения данных и сборки результатов.
- `Mean ≈ 0.5` и `StdDev ≈ 0.2886` соответствуют ожидаемым значениям для равномерного распределения на `[0, 1)`.
- По времени:
  - переход от `np=1` к `np=2` даёт небольшое улучшение;
  - при `np=4` время увеличилось, что типично для сред/задач, где коммуникационные накладные расходы и синхронизация начинают доминировать над вычислениями (особенно в Colab/контейнере).

---

## Вывод
Программа корректно реализует распределённое вычисление среднего и стандартного отклонения с использованием `MPI_Scatterv` и `MPI_Reduce`.  
Измерения показывают, что для `N = 10^6` ускорение ограничено накладными расходами MPI, поэтому увеличение числа процессов не гарантирует уменьшение времени выполнения.
